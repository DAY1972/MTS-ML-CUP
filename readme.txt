Определение пола и возраста владельца HTTP cookie по истории активности пользователя в
интернете на основе синтетических данных
Метрика соревнования — ROC-AUC – для определения пола, f1 weighted – для определения возраста. 
Все решения рассчитываются по формуле -  2 * f1_weighted(по 6 возрастным бакетам) + gini по полу.
Возрастные бакеты( Класс 1 —19-25, Класс 2 —26-35, Класс 3 —36-45, Класс 4 —46-55, Класс 5 —56-65, Класс 6— 66+)

1. eda_final - предварительный анализ данных
1.1. Функционал - анализ предоставленных файлов на наличие NaN значений, пересечение данных в файлах,
создание сводной таблицы по всем файлам с данными, векторизация текстовых переменных
1.2. Входные данные - файлы в формате parquet
1.3. Выходные данные  - векторизованные данные для обучения ML моделей. Данные разделены на два набора по трейн-тест
непересекающиеся. 

По каждому из наборов проводится обучение моделей:
2. MTS_ML_CUP_model_age_1_continuos.ipynb и MTS_ML_CUP_model_age_2_continuos.ipynb
2.1. Функционал - обучение модели прогнозирования возраста, как для непрерывной величины, loss - r2
далее проводим бинаризацию полученных прогнозов посредством MTS_ML_CUP_model_age_1_discret_bins.ipynb и 
MTS_ML_CUP_model_age_2_discret_bins.ipynb
2.2. Входные данные - векторизованные данные для обучения ML моделей
2.3. Выходные данные - обученная модель

3. MTS_ML_CUP_model_age_1_discret_19-71.ipynb и MTS_ML_CUP_model_age_2_discret_19-71.ipynb
3.1. Функционал - обучение модели прогнозирования возраста, как для дискретной величины, loss - CCE
3.2. Входные данные - векторизованные данные для обучения ML моделей
